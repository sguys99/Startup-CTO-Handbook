# IT

여기서 말하는 IT, 즉 정보 기술(information technology)은 회사가 매일 업무를 처리하는 데 쓰는 내부 도구와 기술을 가리킨다. 고객에게 내놓을 제품으로 만들고 있는 기술과는 구분되는 개념이다.

IT는 보통 회사 하드웨어(데스크톱, 노트북, 휴대폰), VPN, 이메일, 백신과 모니터링 소프트웨어 같은 도구로 이루어진다. 요즘 같은 시대의 스타트업이라면, 사무실에 모여 일하든 원격으로 일하든, 몇 가지 현명한 결정만 내리면 IT에 시간이나 자본을 그리 많이 들일 필요가 없다.

대부분의 작은 기술 회사에서 IT 비용을 최소화하는 데 도움이 될 핵심 결정 몇 가지를 짚어보자.

회사 이메일, 데이터, 문서는 클라우드 기반 시스템을 써라. 대다수 스타트업은 구글 워크스페이스(Google Workspace)를 쓰지만, 팀원들(그리고 앞으로 채용할 사람들)이 다른 도구를 더 편하게 여긴다면 그쪽을 택해도 좋다. 이 단계에서 자체 메일 서버, 문서 저장소, 데이터 접근 체계, 네트워크 따위를 직접 구축해서 얻을 이득은 없다.

초기에는 컴플라이언스 시스템상 요구되는 경우가 아니라면 직원에게 회사 하드웨어 사용을 의무화하지 마라. 규모가 작을 때, 특히 제품-시장 적합성을 찾기 전이라면, 회사 하드웨어를 구매하고 지급하고 관리하는 일은 결코 만만치 않은 노력이자 비용인데, 그 대가로 얻는 현실적 이득은 미미하거나 드물다.

인정하기 다소 쓰라릴 수 있지만, 제품과 IT 시스템을 제대로 보호하는 일은 상당히 큰 과제여서 초기 단계의 어린 스타트업이 이를 빠짐없이 해내기란 비현실적이다. 그러니 실용적으로 접근해, 침해나 데이터 유출이 일어날 가능성이 가장 높은 곳, 즉 직원의 인적 실수를 막는 데 집중하길 권한다. 공격자가 데이터를 중간자 공격으로 가로채거나 익스플로잇을 이용해 클라우드 인프라를 해킹하는 일보다, 엔지니어링 팀이 API 앞에 인증을 거는 걸 깜빡하거나 누군가 카페에서 노트북을 잠그지 않고 자리를 비우는 일이 훨씬 더 자주 일어난다.

IT 부담을 최소화하는 모범 사례를 다 따르더라도 피할 수 없는 IT 업무는 여전히 남는다. 주로 사용자 계정을 활성화·비활성화하고 직원의 비밀번호를 복구해 주는 일이다. 이런 작업은 HR 등 다른 동료가 처리할 수 있도록 문서로 정리하고 교육해 두길 권한다. 그래야 당신이나 엔지니어링 팀이 이런 일로 수시로 방해받지 않는다.
